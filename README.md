> ⛔️ **이 저장소를 여는 AI/세션에게 — 절대 규칙**
>
> - 여기 있는 카드 PNG는 **등록된 템플릿(`card_template.html`)으로 파이프라인이 이미 렌더한 최종본**입니다.
> - 브리핑을 요청받으면 이 PNG를 **"그대로 가져와 전달"만** 하세요.
> - **카드를 새로 그리거나 HTML·CSS·디자인·레이아웃을 직접 만들지 마세요.** (재생성/재디자인 금지)
> - 데이터를 새로 뽑아야 하면 카드를 직접 만들지 말고 **파이프라인(GitHub Actions: `Boospatchhao/weekly-meme-briefing`의 `weekly-briefing` 워크플로우)** 을 실행하세요.
> - 디자인을 바꿔야 하면 임의로 하지 말고, 사용자 승인 후 파이프라인의 `card_template.html`을 고쳐 커밋하세요.

## 폴더 구조

- `신고가/latest/` — 최신 **신고가** 브리핑: `briefing.txt` + `card01.png` ~ `cardNN.png` (card01=현황·랭킹, 이후=주요 단지)
- `거래량/latest/` — 최신 **거래량 TOP** 브리핑: 동일 구성
- `latest/` — 가장 최근 실행분(모드 무관, last-wins)
- `weeks/<날짜>-<모드>/` — 주차별 아카이브

## 전달 방법 (요약)

1. `git clone --depth 1 https://github.com/Boospatchhao/weekly-briefing-view.git`
2. 신고가면 `신고가/latest/`, 거래량이면 `거래량/latest/` 폴더의 `card*.png`를 **파일명 오름차순**으로 전부 전달
3. `briefing.txt`는 **원문 그대로** 표시 (톡방 붙여넣기용)

## 카드 디자인 규칙 (참고)

카드는 지역(서울→경기→인천) 순서로 나열되며, 각 지역 안에서 **공급평 ≤37평 먼저 → ≥38평 나중, 각 그룹 가격 높은 순**으로 정렬됩니다. 면적은 공급평(예: `42평`)으로 표기합니다. 이 규칙은 파이프라인(`build_data.py`)에 내장돼 있어 매주 자동 적용됩니다.
